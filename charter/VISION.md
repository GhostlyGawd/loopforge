# VISION — North Star

Loopforge exists to become the best-organized, most trustworthy library of **loop
prompts** in existence: prompts designed to be run repeatedly by Claude Code, where all
memory lives in files and each pass leaves the work one step better.

A visitor should be able to find a loop for anything — "polish this repo," "grow test
coverage," "build this app from a spec," "keep these docs alive" — copy it, run it, and
trust it, because every canonical entry has been built, reviewed, and stress-read by the
system itself.

## What "great" means here
- **Runnable, not aspirational.** Every entry works as pasted.
- **Curated, not hoarded.** A smaller canon beats a large junk drawer. Deprecate freely.
- **Legible.** Taxonomy, schema, and site make the whole collection navigable in seconds.
- **Self-aware.** The system reviews its own output, questions its own charter, and
  records why it changed its mind.
- **Distinctive.** The brand, voice, and site are the system's own — chosen, not defaulted.

## Milestones
- **M1 — Foundation.** Bootstrap complete: tooling green, taxonomy v1, 10 loops
  (≥5 reviewed), brand v1 named and applied, first audit filed.
- **M2 — Working library.** 50 loops; every category populated; ≥60% reviewed or
  canonical; site v2 with search, filters, and per-loop pages; contribution guide.
- **M3 — Reference collection.** 150+ loops across all tiers; canonical set ≥40;
  changelog culture; quarterly audits; the deprecation graveyard tells its own story.

Counts are floors, not goals. Quality ratio (reviewed+canonical ÷ total) is the number
that matters most.
